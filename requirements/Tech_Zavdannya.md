# Технічне завдання
## Фітнес Тренер AI

## 1. Загальна інформація

### 1.1 Назва проекту
**Фітнес Тренер AI** (Fitness Trainer AI)

### 1.2 Мета проекту
Створення веб-додатку для генерації персоналізованих планів тренувань з використанням штучного інтелекту.

### 1.3 Цільова аудиторія
- Люди, які займаються фітнесом
- Початківці у фітнесі
- Досвідчені спортсмени

---

## 2. Функціональні можливості

### 2.1 Автентифікація
- Реєстрація через email та пароль
- Вхід в систему
- Вихід з системи
- Збереження сесії користувача

### 2.2 Профіль користувача
**Параметри:**
- Ім'я, стать, вік, зріст, вага
- Тип тіла, мета тренувань
- Рівень досвіду, частота тренувань
- Цільові групи м'язів

### 2.3 Генерація планів тренувань
- AI генерація через Google Gemini
- Персоналізація під профіль користувача
- Багатоденний розклад
- Детальний опис вправ (підходи, повторення, відпочинок)

### 2.4 Виконання тренувань
- Запуск тренування з таймером
- Покрокове відображення вправ
- Логування підходів (вага, повторення)
- Додавання нотаток
- Збереження результатів

### 2.5 Wellness моніторинг
- Перевірка самопочуття перед тренуванням
- Параметри: енергія, сон, стрес, мотивація
- AI адаптація тренування
- Рекомендації для покращення

### 2.6 AI тренер чат
- Інтерактивне спілкування
- Поради щодо техніки вправ
- Мотиваційні повідомлення
- Контекстуальні відповіді

### 2.7 Аналітика прогресу
- Історія тренувань
- AI аналіз
- Рекомендації для покращення

---

## 3. Технічні вимоги

### 3.1 Технологічний стек

**Frontend:**
- React 18.2.0
- TypeScript 5.3.3
- Tailwind CSS 3.4.1
- Vite 5.0.12

**Backend (BaaS):**
- Firebase Authentication
- Cloud Firestore
- Firebase Realtime Database

**AI:**
- Google Gemini API

**Хостинг:**
- Vercel (production)
- Vercel Edge Network (CDN)
- GitHub (CI/CD)

### 3.2 Структура бази даних

**Firestore колекції:**

```
users/{userId}/profile
├── name, email, age, gender
├── height, weight, bodyType
├── fitnessGoal, experienceLevel
├── trainingFrequency
├── targetMuscleGroups[]
└── healthLimitations[]

workoutPlans/{planId}
├── userId
├── generatedAt
├── goal, duration
└── days[{dayNumber, exercises[]}]

workoutLogs/{userId}/{sessionId}
├── planId
├── startTime, endTime
├── exercises[{sets[], notes}]
└── totalDuration

wellnessChecks/{userId}/{checkId}
├── date
├── energyLevel, sleepQuality
├── stressLevel, motivation
└── painDescription
```

### 3.3 Архітектура

**Тип:** Single Page Application (SPA)

**Структура проекту:**
```
├── components/     # React компоненти
├── hooks/          # Користувацькі хуки
├── services/       # API та бізнес-логіка
├── config/         # Конфігурація Firebase
├── types/          # TypeScript типи
└── utils/          # Допоміжні функції
```

## 4. Нефункціональні вимоги

### 4.1 Продуктивність
- Завантаження сторінки: < 5 секунд
- Генерація AI плану: < 5 хвилин
- Перехід між екранами: < 500 мс
- Синхронізація з Firebase: real-time

### 4.2 Безпека
- HTTPS для всіх запитів
- Firebase Authentication для користувачів
- Firestore Security Rules
- Захист API ключів через змінні середовища
- Шифрування даних при передачі та зберіганні

### 4.3 Масштабованість
- Компонентна архітектура React
- Firebase автоматичне масштабування
- Модульна структура коду
- Code splitting та lazy loading

### 4.4 Зручність використання
- Адаптивний дизайн (mobile, tablet, desktop)
- Інтуїтивний інтерфейс
- Візуальні індикатори прогресу
- Підтримка темної та світлої теми

### 4.5 Надійність
- Uptime > 99%
- Обробка помилок з повідомленнями
- Автоматичне збереження прогресу
- Резервне копіювання через Firebase
